# Agent Plutus

Personal portfolio analysis: daily holdings from Fidelity/Robinhood (SnapTrade), Merrill 401k + cash (Plaid), assembled into `2026_Portfolio_Analysis.xlsx`.

## Read first

- `skills.md` — capability map (which script does what, inputs/outputs, how to invoke). Read this before exploring the tree.
- `tests/README.md` — testing philosophy and traceability table.

## Rules

- **Live API side effects.** Running `daily_pipeline.py` or any extractor hits SnapTrade/Plaid and writes to `extract_output/`, `data/`, and `snapshots/`. Don't run them casually for inspection — use `--dry-run` or read existing JSON instead.
- **Flat layout is live.** `<project>/*.py` is what the scheduler runs; `repo/` is canonical source but not scheduled. Edits to `repo/` won't take effect on the next pipeline run.
- **Path resolution lives in `paths.py`.** Never hardcode `~/OneDrive/...` or `C:\Users\...`. Add new path constants there or read from `~/.portfolio_extract/config.json`.
- **Don't hand-edit `registry_data.json`.** It's regenerated by rebuild scripts; manual edits cause validator drift and false "data bugs."
- **`manual_data.json` is hand-maintained.** Angel valuations and 401k balances live there. The pipeline reads but does not overwrite — except `run_angel_check.py`, which prompts for user approval before writing.
- **Python:** Windows env, use `python` not `python3`.
- **Credentials:** `~/.portfolio_extract/config.json` and `<project>/.anthropic_key`. Never commit, print, or paste into chat tools.

## Testing

- **Bug-fix workflow:** write a failing test that reproduces the bug *before* fixing. Commit test + fix together — never ship a fix without a test.
- **Naming:** `test_<condition_that_must_hold>`, not `test_<bug_description>`. Tests read as specs.
- **Run:** `python -m pytest tests/ -v` from project root (~8 seconds for all 108 tests).
- **Synthetic fixtures only.** Don't commit real holdings, account numbers, or cost bases.
- **No CI, no coverage gate, no mocked network tests** — this is intentional, don't add them.

## Workflow norms

- After workbook changes: `python validate_workbook.py` — non-zero exit means structural breakage.
- Logs land in `logs/pipeline_*.log` per run.
- For "did the workbook actually look right" checks, open the .xlsx — validator only catches structural issues.
